# P1. Data Collection 규정

## 전체 data set 규정
- ChEMBL 36 데이터 이용
- Organism: human, mouse, rat만 포함
- 분자(리간드) 크기: 중원자(heavy atoms) 수가 80개 이하인 저분자 화합물만 포함
- 단백질(타겟): Target Type이 `SINGLE PROTEIN` 또는 `PROTEIN COMPLEX`인 경우만 포함
- Positive data 1개당 위 규칙에 따른 10개 Negative data를 한 세트로 구분할 수 있는 태깅 추가(`pair_id`)
- OPREA-1 기준으로 화합물 Scaffold 정보를 추출할 수 있는 코드 포함, 각 화합물 정보에 `scaffold_smiles` 추가
- 10-fold 교차 검증을 위해 폴드 번호 부여하고 가능한 데이터 편중이 발생하지 않도록 분배(`cv_fold`)

## Positive data set
- 직접 결합: 활성 유형(Assay type)이 `B`(Binding)로 명시된 직접적인 결합 데이터만 사용
- 고친화도: 활성 값(`Ki`, `Kd`, `IC50`, `EC50`)이 10 μM 미만인 경우만 활성(Active)으로 간주
- 모호성 제거: 동일한 상호작용에 대해 10 μM 미만의 활성과 그 이상의 비활성 데이터가 혼재하는 모호한 데이터는 훈련셋에서 제외

## Negative data set
- 저친화도: 활성 값(`Ki`, `Kd`, `IC50`, `EC50`)이 500 μM 미상인 경우만 비활성(Non-active)으로 간주 (2013년도 기준, 1799 interactions involving 1214 molecules)
- 저친화도 보강: 조건1) 한 타겟 당 10개 Negative data가 포함되도록 non-interacting ligand–protein pairs 중에서 랜덤 선택하고, 쿼리 분자가 해당 타겟의 파라로그(Paralog) 단백질(BLAST E-value ≤10^-5)과 상호작용한다고 알려진 경우, 이 쌍은 음성 데이터에서 배제
- 활성 값이 10~100 μM 사이(약한 결합)인 데이터가 존재하는 경우도 제외

## 생성되는 최종 산물 파일
- Parquet 파일: `chembl36_stp_training_set`
- CSV 파일: Parquet 파일에 포함된 정보 리스트 및 예시 1000건(랜덤 생성)
- 화합물(Ligand) 관련 정보: `molregno`, `InChIKey`, `mol_chembl_id`, `canonical_smiles`, `heavy_atoms`, `scaffold_smiles`
- 타겟(Protein) 관련 정보: `target_chembl_id`, `target_name`, `target_type`, `organism`, `uniprot_id`, `sequence`
- 약리 활성(Activity) 관련 정보: `standard_value`, `standard_type`, `standard_units`, `activity_uM`(μM 단위로 표준화된 활성 수치), `label`(학습용 라벨, Active: 실제 활성이 확인된 데이터, Assumed_Inactive: 네거티브 샘플링을 통해 비활성으로 간주된 데이터)
- 파이프라인 및 학습용 메타데이터: `publication_year`, `confidence_score`, `source_version`, `set_type`(데이터의 성격: Positive는 실제 실험 데이터, Negative는 생성된 비활성 데이터), `cv_fold`(10-fold 교차 검증을 위해 할당된 폴드 번호 1~10)
